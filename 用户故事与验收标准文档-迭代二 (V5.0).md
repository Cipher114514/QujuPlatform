# 用户故事与验收标准文档-迭代二 (V5.0 )

### US-014-A: 地图模式活动分布与详情闭环

- **用户故事描述**：作为一个用户，我希望能在地图上直观查看到附近的线下活动分布，以便根据地理位置就近参与。
- **验收标准**：
  - [ ] 接入高德地图 SDK，在前端页面渲染出带有标记点（Marker）的地图视图。
  - [ ] 点击标记点需能轻量化弹出该活动的摘要信息卡片，并可点击跳转至活动详情页。
  - [ ] 异常处理：在未授权定位或网络延迟时，应降级展示或提示加载失败。

### US-016-A: 活动报名、取消与排队履约流

- **用户故事描述**：作为一个用户，我希望能够报名感兴趣的活动，并在报名截止前取消，若满额则能自动进入候补队列，以便灵活安排行程。
- **验收标准**：
  - [ ] 点击报名扣减名额，状态机流转为“已报名”；取消报名时名额即时释放。
  - [ ] 采用 FIFO 队列算法，报名满额后自动进入排队状态，名额释放时队首用户自动转正。
  - [ ] 超过活动截止时间，报名/取消操作按钮置灰。

### US-019-A: 现场签到与核销闭环机制

- **用户故事描述**：作为活动发起者，我希望能够使用动态二维码对参与者进行现场核销签到，以便防范代签风险。
- **验收标准**：
  - [ ] 系统为每场活动生成定时刷新（如30秒）的动态核销二维码。
  - [ ] 发起者端扫描参与者出示的二维码，校验通过后状态同步更新为“已签到/已核销”。
  - [ ] 边界：已失效、过期或重复核销的二维码需拦截并提示失败。

### US-021-A: 活动评价、花絮展示与数据复盘

- **用户故事描述**：作为一个用户，我希望活动结束后能对活动进行星级评价；发起者能上传图文花絮并查看活动复盘统计。
- **验收标准**：
  - [ ] 提供 1-5 星评分组件及评价文本输入，提交后关联至活动详情页展示。
  - [ ] 允许发起者上传多张图片和总结文字作为活动花絮。
  - [ ] 后台记录并展示活动的最终参与人数、签到率等基础数据看板。

### US-024-A: 好友关系与社交互动体系

- **用户故事描述**：作为一个用户，我希望能够关注其他用户并建立好友关系，同时通过即时私聊进行一对一沟通。
- **验收标准**：
  - [ ] 实现关注/粉丝逻辑，个人中心展示关注与粉丝列表数据统计。
  - [ ] 集成 WebSocket 实现即时私聊，支持文字实时转发及本地落库存储。

### US-027-A: 兴趣小队全生命周期管理

- **用户故事描述**：作为一个用户，我希望能够创建或加入兴趣小队，并通过专属空间进行交流、活动发布与管理。
- **验收标准**：
  - [ ] 支持小队创建、按标签发现、申请准入及队长管理成员。
  - [ ] 小队创建后自动生成专属群聊空间，支持共享相册、队内活动发布及一键强制解散（仅队长权限）。

### US-038-A: 平台运营管理与风控审批体系

- **用户故事描述**：作为一个平台管理员，我希望通过独立后台全面管理账号状态、资质审核及违规风控，以维护平台生态。
- **验收标准**：
  - [ ] 提供独立后台入口，实现全平台个人/商家账号分页检索与“封禁/解封”管理。
  - [ ] 设立“商家审批看板”，支持资质凭证图片审核；提供违规活动“一键下架”与资质追溯功能。

### US-007-A: [AI] 智能内容辅助策划系统

- **用户故事描述**：作为一个活动发起者，我希望系统能根据输入的主题自动生成活动策划案，以便提高内容创作效率。
- **验收标准**：
  - [ ] 接入大模型 API，提供精简 Prompt 模板（输入主题/类型/时长，输出策划大纲）。
  - [ ] 生成内容支持在前端富文本框中进行二次编辑与保存。

### US-010-A: [AI] 敏感内容过滤与图片特征标签化

- **用户故事描述**：作为平台方，我希望发布的内容在入库前能被 AI 自动识别并过滤敏感词，同时对图片进行自动打标。
- **验收标准**：
  - [ ] 在活动发布/留言提交环节调用接口进行敏感词检测，违规直接拦截并提示。
  - [ ] 上传活动配图时后台自动调用视觉识别接口，提取并保存不少于 3 个特征标签。

### NFR-001-A: 核心性能、安全与容灾保障

- **用户故事描述**：系统核心功能必须具备高响应、高并发处理能力及数据安全性。
- **验收标准**：
  - [ ] 核心接口（首页、搜索、详情）在标准网络下 95% 响应时间低于 1000ms。
  - [ ] 核心报名/抢单场景集成 Redis 分布式锁，严防超卖；采用 HTTPS 传输与加盐哈希存储密码，并对关键接口进行 JWT 鉴权加固。

### [选做模块] 迭代二增强功能

#### US-032-M: 小队深度管理与贡献度体系

- **用户故事描述**：作为一个队长/小队成员，我希望能够精细化管理小队权限（管理成员/管理员），并参与活跃度积分排名，以便实现小队的自治与良性竞争。
- **验收标准**：
  - [ ] **权限体系**：队长可灵活授予/撤销成员的“管理员”权限，管理员获得审核准入、管理公告与活动等子权限。
  - [ ] **积分体系**：系统自动计算成员的贡献值（如签到、参与活动），生成队内积分排行榜，体现成员贡献度。
  - [ ] **管控闭环**：平台管理员后台支持对违规小队进行“停用”或“彻底封禁”，操作后前端小队不可见且群聊自动解散。

#### US-036-M: 私聊交互体验优化（撤回与已读回执）

- **用户故事描述**：作为一个即时通讯用户，我希望在私聊中能撤回失误发送的消息并查看已读状态，以便提升沟通效率与体验。
- **验收标准**：
  - [ ] **消息撤回**：支持发送后 2 分钟内的消息撤回功能，撤回后原内容移除并显示“你撤回了一条消息”提示。
  - [ ] **已读回执**：在对话界面针对单条消息集成 ACK 报文机制，对方查看消息时状态实时同步为“已读”。
