### 4.1 模块间连接

```
P1 #/create-activity ──────────────────────────────────────── 独立
P2 #/ai-create ──→ 发布后跳 #/activity/:id ──→ P4 的详情页
    #/my-activities ──→ 点击跳 #/activity/:id ──→ P4 的详情页
P3 #/activities ──→ 点击卡片跳 #/activity/:id ──→ P4 的详情页
P4 #/activity/:id
    ├── 已报名 → 链接跳 #/my-registrations ──→ P5 的管理页
    ├── 已满员 → 链接跳 #/activity/:id/waitlist ──→ P5 的队列页
    └── 发起人 → 链接跳 #/chat/:userId ──→ P7 的聊天
P5 #/my-registrations ──→ 点击跳 #/activity/:id ──→ P4 的详情页
    #/activity/:id/waitlist ──→ 返回 #/activity/:id
P6 #/friends ──→ 发消息跳 #/chat/:userId ──→ P7 的聊天
P7 #/chat ──→ 点击跳 #/chat/:userId ──→ 独立
P9 #/discover ──→ 点击用户跳 #/follows ──→ 关注/粉丝管理
    #/follows ──→ 发消息跳 #/chat/:userId ──→ P7 的聊天
```

**模块间仅通过 `<a href="#/xxx">` 超链接跳转。不修改别人的 JS 文件。**

## 六、开发规范

### 6.1 通用规则

1. **每个人只修改自己负责的文件**，不碰别人的 JS/CSS/Java
2. **模块间仅通过 `<a href="#/xxx">` 跳转**，不注入 DOM
3. **前端 mock 先行**：每个页面内置 `USE_MOCK = true` 开关，后端未好也能开发
4. **API 契约优先**：接口格式严格参照本文档第三节，响应用 `Result<T>` 包装
5. **每日 push**：代码至少每天推一次，避免最后一天冲突

### 6.2 接入步骤

```bash
# 1. 拉代码
git clone ... && cd demo/demo

# 2. H2 开发模式（不需 MySQL）
.\gradlew.bat bootRun

# 3. 浏览器 http://localhost:8080
#    登录: user@test.com / test1234

# 4. 创建页面: static/js/pages/xxx.js
#    模板参考: README-新人接入指南.md

# 5. 在 static/index.html 加 <script> 引入

# 6. 开发时 USE_MOCK = true，联调时改 false
```
